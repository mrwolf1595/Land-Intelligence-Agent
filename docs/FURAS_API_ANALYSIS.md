# تحليل منصة فرص (Furas API Analysis)
**تاريخ التحليل:** أبريل 2026

بناءً على طلبك باستكمال باقي المنصات، أجرينا تحليلاً دقيقاً لآلية عمل بوابة "الخارطة الاستثمارية" على منصة **فرص** التابعة لوزارة البلديات.

---

## 1. اكتشاف الواجهة الخلفية
المنصة مبنية بنظام إدارة المحتوى المتطور (Drupal)، وتستخدم تقنيتين رئيسيتين لجلب البيانات للمستخدم مما يوفر خيارين لاستخراج البيانات (Scraping):

### الأسلوب الأول: واجهة الخريطة المباشرة (JSON Endpoint)
* **المسار:** `GET https://furas.momah.gov.sa/ar/api/map/opportunities`
* **الاستخدام:** هذا هو المسار الـ "أنظف" للذكاء الاصطناعي الخاص بنا. بمجرد استدعائه، يرجع لك الخادم مصفوفة ضخمة جداً تحتوي على النقاط الجغرافية للاستثمارات (عبر الخريطة).
* **محتوى الرد (Response):** يحمل تفاصيل مثل `lat` و `lon`، ونوع النشاط (`activity`) والجهة المالكة كالأمانة والبلدية. مناسب جداً لرسم خرائط حرارية للتنافسية.

### الأسلوب الثاني: واجهة كروت الفرص (Drupal AJAX)
* **المسار:** `GET https://furas.momah.gov.sa/ar/opportunities-listing/Investment?_wrapper_format=drupal_ajax`
* **الاستخدام:** عند الضغط على "تصفية" يبدأ هذا المسار بالعمل. وبدلاً من أن يعيد صفحة ويب، فإنه يعيد مصفوفة من أوامر دروبال `commands` بداخلها الـ HTML المحدث لكروت الفرص (الأوصاف التفصيلية، المساحات كـ 4200 متر مربع، وتاريخ الشراء). يحتاج بارسر `BeautifulSoup` لاستخراج البيانات من داخل الـ JSON.

---

## 💡 التوصية لتعديل `furas/scraper.py`:
استخدم **الأسلوب الأول** لتتبع المشاريع البلدية القريبة من قطعة أرض العميل لمعرفة "جاذبية الحي الاستثمارية" ضمن نطاق (radius_km). إذا وجدت أن البلدية تطرح مشروع حديقة عامة أو مول تجاري ضخم بالقرب من أرض سكنية للعميل، سيقوم Agent التحليل بكتابة تعليق إيجابي (Competitive Impact: Positive) ويرفع مؤشر التقييم!
