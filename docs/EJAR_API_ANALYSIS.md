# تحليل شبكة مؤشر إيجار (Ejar Network Analysis)
**تاريخ التحليل:** أبريل 2026

بناءً على طلبك بفتح المتصفح فعلياً والقيام بمراقبة الطلبات الشبكية (Network Requests) بدلاً من كتابة الـ Scraper بشكل أعمى، قام وكيل المتصفح (Browser Subagent) بتجاوز واجهة الموقع وتحقيقات الحماية والوصول للـ API الخلفية التي تغذي لوحات "مؤشر إيجار".

النتيجة كانت ممتازة! الـ API الحقيقي يعمل بشكل مباشر ولا يحتاج للتعامل مع تعقيدات واجهة الموقع الأمامية المعرّضة للحماية بـ Cloudflare.

---

## 1. المعلومات الأساسية للـ API المباشر (Base Info)
* **رابط الأساس (Base URL):** `https://rentalrei.rega.gov.sa/RegaIndicatorsAPIs/api/IndicatorEjar/`
* **المسؤول (Host):** الهيئة العامة للعقار (REGA).
* **نوع المحتوى (Content-Type):** `application/json`
* **المصادقة (Auth):** المذهل هو أن الـ API لا يتطلب Token حقيقي للاستعلامات العامة. يمكن إرسال الترويسة `Authorization: Bearer null`.

---

## 2. المسارات (API Endpoints) المستخرجة

### أ. مسارات أدلة البيانات (Lookups)
يجب استخدام هذه الاستعلامات للحصول على الرموز الرقمية (IDs) للمدن والأحياء، بدلاً من استخدام الأسماء النصية:
* **جلب كل المناطق:** `GET /GetAllRegions`
* **جلب المدن بمنطقة معينة:** `GET /GetCitisByRegionId?regionId={مؤشر_المنطقة}` (مثال: منطقة الرياض تعادل `1`).
* **جلب أحياء مدينة معينة:** `GET /GetDistrictsByCityId?cityId={مؤشر_المدينة}` (مثال: مدينة الرياض تعادل `21282`).
* **أنواع الوحدات:** `GET /GetAllResidentialUnitsTypes` (يحدد نوع الوحدة السكنية أو التجارية).

### ب. مسارات البيانات والمؤشرات (Data Indicators)
* **تفاصيل المنطقة المستهدفة (أهم نقطة لنا):** `POST /GetDetailsV2`
* **المخططات البيانية (Charts):** `POST /GetChartPublicIndicatorsDemo`
* **مؤشرات عامة (العقود النشطة بالمدينة):** `POST /GetPublicIndicatorDemo`

---

## 3. هندسة الطلب الأساسي (Request Payload) - `GetDetailsV2`
للحصول على بيانات دقيقة (أسعار وعدد صفقات) لحي معين، يجب إرسال طلب `POST` للجسم (Body) التالي:

```json
{
  "trigger_Points": "0",          // استخدم "0" للبحث عن مستوى المدينة بالكامل، أو معرف الحي للتخصيص
  "cityId": 21282,                // معرف المدينة (مثال 21282 = الرياض)
  "regionId": 1,                  // معرف المنطقة (مثال 1 = المنطقة الوسطى)
  "unitTypeId": 1,                // نوع الوحدة المستهدفة (مثال 1 = شقة)
  "rentalUnitUsage": 0,           // 0 للاستخدام السكني، 1 للاستخدام التجاري
  "strt_date": "2024-03-31T21:00:00.000Z",
  "end_date": "2025-02-28T21:00:00.000Z",
  "isQuarterly": 0                // 0 للبيانات المجمعة السنوية، 1 لتوزيعها الربع سنوي
}
```

---

## 4. هندسة الاستجابة (JSON Response Struct)
سيرجع الخادم مصفوفة (Array) تحتوي على هيكل كل حي/فترة زمنية، وأهم المفاتيح (Keys) التي تهمنا هي:
* `sumDeals`: إجمالي الصفقات التي تمت فعلياً.
* `sumRent`: إجمالي القيمة الإيجارية للحي.
* `avgRent`: متوسط الإيجار (يفيدنا جداً لـ CAP Rate).
* `avgArea`: متوسط مساحات الوحدات المؤجرة.
* `highestRent` / `lowestRent`: أعلى وأقل إيجار مسجل ضمن النطاق المطلوب.

---

## 💡 التوصية لتعديل كود `ejar/scraper.py` لاحقاً:
عندما تبدأ البرمجة، لن تحتاج لـ `Selenium` أو `Playwright`! نظراً لاكتشافنا واجهة הـ API المباشرة المفتوحة من REGA (بواسطة مراقبة الـ Network Requests)، يمكنك كتابة دالة `requests.post` مباشرة في بايثون تمرر الـ Payload المذكور، وستحصل على أدق البيانات بأقل استهلاك لموارد السيرفر (بدون أن يحظرك حارس الموقع Cloudflare).
