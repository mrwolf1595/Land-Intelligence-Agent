# تحليل البنك المركزي السعودي - ساما (SAMA SAIBOR API)
**تاريخ التحليل:** أبريل 2026

بناءً على التكليف بإنهاء القائمة، وجهنا الوكيل لتحليل واجهات البنك المركزي السعودي (SAMA) للحصول على أرقام "السايبور" (SAIBOR) والفائدة بشكل تقني (حيث تعتبر هذه الأرقام هي المحرك الأساسي لحسابات الـ CFO وتكاليف الفرصة البديلة في العقار).

---

## 1. الوصول للبيانات وتحديد الهدف
تم تجاوز واجهة الموقع الإخبارية والولوج إلى قسم **بوابة الإحصاءات التفاعلية (Open Data Portal)** داخل موقع (ساما).
وقد ركز الوكيل على قسم الفائدة وتحديداً:
`Categories > Exchange Rates and Interest Rates > Monetary Indicators: (Interest Rates)`
ثم قام باختيار مؤشر الفائدة المرجعي للعقارات: **Monthly Avg. 3M SAIBOR**.

---

## 2. كيفية عمل واجهة الـ API (Network Analysis)
بعد تفعيل توليد الـ API وعرض المخطط، ظهرت تفاصيل الطلب التقني (API Specs):
* **رابط الاستعلام الأساسي:**
  `POST https://www.sama.gov.sa/_layouts/15/SAMA.Internet.WCM/WebMethods.aspx/GetStatistcalById`
* **معاملات الإرسال (Query Params):**
  المؤشر الخاص بـ 3M SAIBOR يحمل الرقم التعريفي `id=3259` واللغة `lang="en"`.
* **محتوى الترويسة (Headers):**
  الخادم مبني بتقنية (ASP.NET WebMethods)، لذلك من الضروري إرسال ترويسة:
  `Content-Type: application/json`

---

## 3. محتوى الاستجابة (JSON Response)
البيانات المسترجعة عبارة عن كائنات JSON مسلسلة، تحتوي على عمودين رئيسيين:
* **التاريخ (Date):** (مثال: January 2026).
* **القيمة (Value):** (نسبة السايبور، مثال: 5.82).

*(البيانات دقيقة ومحدثة شهرياً وتمتد لسحب الهيستوري الخاص بالفائدة من 2007 وحتى اللحظة الحالية للطلب).*

---

## 💡 التوصية لتعديل `sama/scraper.py` لاحقاً:
لا تقم ببناء متصفح آلي (Selenium) لـ SAMA لأن موقعهم يقدم هذه الـ API صراحةً لتسهيل الربط.
عوضاً عن الـ `Mock Data` الموجودة حالياً، ستكتب دالة `requests.post` بسيطة تستهدف الرابط أعلاه بالـ `id=3259`، وتأخذ "أول/أحدث" قيمة في المصفوفة العائدة لتمريرها مباشرة كـ `debt_rate` في محرك `financial.py`. هذا سيجعل النظام متصلاً بالاقتصاد الكلي السعودي بالوقت الفعلي بدقة خرافية!
